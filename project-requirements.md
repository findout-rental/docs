# Project Requirements

## Application Description

This project aims to develop an application for booking residential apartments, allowing users to search for accommodation for a specific period and view its details.

In recent years, accommodation booking applications have become an essential part of the modern travel experience, enabling users to easily and quickly find suitable apartments or rooms via smartphone.

The project provides a comprehensive practical experience that includes the basic tasks for building a commercial booking application, including registration and login, searching for apartments, viewing details, completing the booking process, reviewing previous bookings, in addition to setting up a personal profile.

## Note

It is sufficient to complete the booking process with the necessary information (location and payment information) for it to be approved by the server only.

## Mandatory Requirements (9 marks)

1. Allow the user to register within the mobile application via mobile number, whether as an apartment owner or a tenant (2 marks).

2. Compel tenants and apartment owners to enter their personal information (2 marks):
   a. First Name.
   b. Last Name.
   c. Personal Photo.
   d. Date of Birth.
   e. ID Photo.

3. Provide the user with the ability to log in and out of the mobile application (2 marks).

4. Provide the user with the ability to browse apartments and view their specifications (3 marks).

## Note

The registration request for apartment owners and tenants must be reviewed by the System Administrator (Admin) before allowing them to use the application.

## Basic Requirements (9 marks)

1. Provide the tenant with the ability to filter apartments by governorate, city, price, and according to specific specifications (2 marks).

2. Allow multiple users to book an apartment for a specific period without booking conflicts (2 marks).

3. Provide the user with the ability to cancel or modify a booking (2 marks).

4. Provide the user with the ability to view all bookings they have made, both previous and current, including cancelled ones (2 marks).

5. Provide the tenant with the ability to rate apartments they have previously booked (1 mark).

## Note

The apartment owner must approve booking requests before the accommodation is considered booked, and they must also approve modification requests.

## Additional Requirements

1. Implement a notification system where the system sends a notification to the user for every change in their booking status (2 marks).

2. Ability to change the language to Arabic/English for the customer mobile application and admin web application (changing texts and changing page direction RTL/LTR) (2 marks).

3. In-app messaging system allowing the tenant and apartment owner to communicate with each other from within the application (2 marks).

4. Ability to change the application mode between dark and light mode (Light/Dark Mod) (2 marks).

5. Provide the tenant with the ability to add apartments to a favorites list, with the option to perform the booking process from the favorites list (1 mark).

## Note

A Flutter web application must be built for the System Administrator (Admin) to approve registration requests and delete users from the application if necessary.
